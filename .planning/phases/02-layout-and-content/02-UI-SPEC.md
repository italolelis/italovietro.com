## Book Shortcode Contract

The `{{< book >}}` shortcode is the core deliverable of this phase. All book/newsletter/podcast entries are rendered through it.

### Parameters

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| `title` | string | yes | Book or resource title |
| `author` | string | yes | Author name(s) |
| `link` | string | yes | URL (Amazon, Substack, show page) |
| `type` | string | yes | One of: `book`, `newsletter`, `podcast` |
| `rating` | string | no | Reserved for Phase 3 — accept parameter but do not render |

The description / personal note is passed as the shortcode body (inner content).

### Rendered HTML Structure

```html
<div class="book-entry book-entry--{type}">
  <div class="book-entry__header">
    <h4 class="book-entry__title">
      <a href="{link}" target="_blank" rel="noopener noreferrer">{title}</a>
    </h4>
    <span class="book-entry__author">{author}</span>
  </div>
  <div class="book-entry__description">
    {inner content / personal note}
  </div>
  <!-- .book-entry__rating slot reserved for Phase 3 — absent in Phase 2 -->
</div>
```

**Phase 3 extensibility:** The `.book-entry__rating` slot is intentionally absent in Phase 2. Its absence must not create a layout gap. The `__header` and `__description` blocks should stand complete without it.

`layouts/shortcodes/book.html`

## Section and Tier Structure

### Currently Reading Section

- Placed at the top of the page, before all category sections.
- Uses the same `{{< book >}}` shortcode.
- Section heading: `## Currently Reading`
- No special wrapper beyond the standard heading + shortcode entries.
- 1–3 entries at any time; the personal note (body content) serves as "why I'm reading this."

### Category Sections

Each category uses `## Category Name` as its heading. Tiers within each category use `### Tier Label` as sub-headings.

**Standardized tier vocabulary** (source: 02-CONTEXT.md D-08):

| Tier | Heading text |
|------|-------------|
| 1 (highest) | Must Read |
| 2 | Recommended |
| 3 | Worth Your Time |

These replace the current inconsistent "Essential" / "Highly Recommended" labels.

### Category Visual Separation

Categories are separated by spacing alone — no decorative dividers between categories. The `## h2` heading with its border-bottom (already styled in Phase 1 `_reading-list.scss`) provides sufficient visual weight to start each new category. A `margin-top: 3rem` above each `h2` (already in Phase 1) is preserved.

Tier sub-headings (`### h3`) are styled as subtle, typographic labels — not structural headings. They carry visual weight through font-size and color contrast, not decoration.

## Spacing Scale

Inherited from Phase 1 contract. No new tokens introduced.

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding |
| sm | 8px | Compact spacing — book entry author line margin |
| md | 16px | Default element spacing — gap between title and description |
| lg | 24px | Section padding — padding within book entry block |
| xl | 32px | Layout gaps |
| 2xl | 48px | Major section breaks — spacing above category h2 |
| 3xl | 64px | Page-level spacing |

**Book entry specific spacing:**
- `.book-entry` padding: 0 (no card padding — typography-driven, no borders)
- `.book-entry__title` margin-bottom: 4px (xs)
- `.book-entry__author` margin-bottom: 8px (sm)
- `.book-entry__description` margin-bottom: 0
- Gap between book entries within a tier: 24px (lg)
- Gap between tier heading and first entry: 12px (between sm and md — acceptable exception)
- Gap between last entry in one tier and next tier heading: 32px (xl)

Exceptions: 12px gap between tier heading and first entry (not on the 8-point grid — closest values are 8px and 16px, both too narrow/wide for this context).

## Typography

All values continue from Phase 1. Phase 2 adds only the book entry title and author roles.

| Role | Size | Weight | Line Height | Element |
|------|------|--------|-------------|---------|
| Body | 16px (`$global-font-size`) | 400 | 1.5 | Page prose, entry descriptions |
| Secondary / author | 14px (0.875rem) | 400 | 1.5 | `.book-entry__author`, tier labels (h3) |
| Book entry title | 17px (1.0625rem) | 600 | 1.3 | `.book-entry__title a` |
| Tier label (h3) | 13px (0.8125rem) | 600 | 1.2 | `### Must Read`, `### Recommended`, etc. |
| Section heading (h2) | 32px (2rem) — preserved from Phase 1 | 700 | 1.2 | Category headings |
| Intro paragraph | 17.6px (1.1rem) — preserved from Phase 1 | 400 | 1.8 | First paragraph |

**Weight scale: [400, 600] only.** Phase 1 used 700 for h2 (LoveIt default). Phase 2 introduces 600 (semibold) for book titles and tier labels. No new weights beyond these three.

**Tier label visual treatment:** All-caps, letter-spacing 0.08em, font-size 0.8125rem, weight 600, color `$global-font-secondary-color` (light) / `$global-font-secondary-color-dark` (dark). This makes tier labels read as metadata, not content headings.

## Color

Inherited from Phase 1 contract. Phase 2 adds secondary text color for author lines and tier labels.

| Role | Light Value | Dark Value | Usage |
|------|------------|-----------|-------|
| Dominant (60%) | `#fff` (`$global-background-color`) | `#292a2d` (`$global-background-color-dark`) | Page background |
| Secondary (30%) | `#f0f0f0` (`$global-border-color`) | `#363636` (`$global-border-color-dark`) | h2 border-bottom, hr dividers |
| Accent (10%) | `#2d96bd` (`$single-link-color`) | `#55bde2` (`$single-link-color-dark`) | Book entry title link color only |
| Text primary | `#161209` (`$global-font-color`) | `#a9a9b3` (`$global-font-color-dark`) | h2, h3, body prose |
| Text secondary | `#a9a9b3` (`$global-font-secondary-color`) | `#5d5d5f` (`$global-font-secondary-color-dark`) | Author line, tier labels |
| Destructive | not applicable | not applicable | No destructive actions in Phase 2 |

**Accent reserved for:** book entry title link text color only. Not used for tier labels, author lines, borders, or backgrounds. No hover border-bottom on entry titles (clean typography-driven approach — no underlines except on hover via standard browser behavior).

**Book entry background:** none (transparent). Phase 1's `$global-background-color` background on `ul > li` is removed in Phase 2 as part of eliminating the card design. Book entries render directly on the page background.

**Borders:** none on book entries. Phase 1's `border: 1px solid` and `border-radius: 12px` on `ul > li` are removed.

## Interaction Contract

Phase 2 removes the card hover effects from Phase 1 (translateY, box-shadow, border-color change). The design is typography-driven with no interactive decoration.

| Interaction | Behavior |
|------------|----------|
| Book title link hover | `color` transitions to `$single-link-hover-color` (`#ef3982` light / `#bdebfc` dark) — standard LoveIt link hover |
| Card hover (translateY + box-shadow) | REMOVED — no cards in Phase 2 |
| List item hover border-color change | REMOVED — no borders in Phase 2 |
| Anchor nav pill hover | Preserved — Phase 3 will replace anchor nav entirely |
| All other interactions | None added |

**Transition:** Book title link color uses `transition: color 0.2s ease`.

## Shortcode SCSS Contract

New styles are appended to `assets/css/_reading-list.scss`. The Phase 1 card styles (`ul > li` background, border, box-shadow, hover) are replaced by shortcode-specific selectors.

### Light mode additions

```scss
// Book entry — typography-driven, no cards
.single .content .book-entry {
    margin-bottom: 24px; // lg
}

.single .content .book-entry__title {
    font-size: 1.0625rem;
    font-weight: 600;
    margin-bottom: 4px; // xs
    line-height: 1.3;
}

.single .content .book-entry__title a {
    color: $single-link-color;
    text-decoration: none;
    transition: color 0.2s ease;
}

.single .content .book-entry__title a:hover {
    color: $single-link-hover-color;
}

.single .content .book-entry__author {
    font-size: 0.875rem;
    color: $global-font-secondary-color;
    margin-bottom: 8px; // sm
    display: block;
}

.single .content .book-entry__description {
    font-size: 1rem;
    line-height: 1.5;
    color: $global-font-color;
}

// Tier labels (h3 within category sections)
.single .content h3 {
    font-size: 0.8125rem;
    font-weight: 600;
    letter-spacing: 0.08em;
    text-transform: uppercase;
    color: $global-font-secondary-color;
    margin-top: 32px; // xl
    margin-bottom: 12px;
}
```

### Dark mode additions (same pattern as Phase 1)

```scss
[theme=dark] .single .content .book-entry__title a {
    color: $single-link-color-dark;
}

[theme=dark] .single .content .book-entry__title a:hover {
    color: $single-link-hover-color-dark;
}

[theme=dark] .single .content .book-entry__author {
    color: $global-font-secondary-color-dark;
}

[theme=dark] .single .content .book-entry__description {
    color: $global-font-color-dark;
}

[theme=dark] .single .content h3 {
    color: $global-font-secondary-color-dark;
}
```

Auto theme dark mode mirrors the above under `@media (prefers-color-scheme: dark) { [theme=auto] ... }`.

### Styles to remove / replace from Phase 1

The following Phase 1 selectors target the old `ul > li` card pattern and must be removed or overridden when shortcode markup replaces the raw markdown lists:

- `.single .content > ul > li { background, border, border-radius, box-shadow, transition }`
- `.single .content > ul > li:hover { transform, box-shadow, border-color }`
- `.single .content > ul > li strong { ... }`
- `.single .content > ul > li a { ... }`
- `.single .content > ul > li a:hover { ... }`
- All dark/auto equivalents of the above

These selectors become inert once shortcode markup replaces raw `ul > li` in the markdown content, but they should be cleaned up in the same PR to keep the SCSS minimal.
