  <action>
Create `layouts/shortcodes/book.html` with this exact content (per D-01, D-02, D-03):

```html
{{- /* layouts/shortcodes/book.html */ -}}
{{- /* Parameters: title (required), author (required), link (required), type (required), rating (reserved Phase 3) */ -}}
{{- $title  := .Get "title" -}}
{{- $author := .Get "author" -}}
{{- $link   := .Get "link" -}}
{{- $type   := .Get "type" | default "book" -}}
{{- $inner  := trim .Inner "\n" | .Page.RenderString -}}
<div class="book-entry book-entry--{{ $type }}">
    <div class="book-entry__header">
        <h4 class="book-entry__title">
            <a href="{{ $link }}" target="_blank" rel="noopener noreferrer">{{ $title }}</a>
        </h4>
        <span class="book-entry__author">{{ $author }}</span>
    </div>
    <div class="book-entry__description">
        {{- $inner -}}
    </div>
    {{- /* .book-entry__rating slot reserved for Phase 3 — absent in Phase 2 */ -}}
</div>
```

Key design decisions:
- `trim .Inner "\n"` strips leading/trailing newlines before `.Page.RenderString` to avoid extra `<p>` wrapping (Research Pitfall 1)
- `$type` defaults to `"book"` but accepts `"newsletter"` and `"podcast"` for BEM modifier classes `book-entry--newsletter`, `book-entry--podcast`
- `rating` parameter is NOT declared — Phase 3 will add it. The HTML comment marks where the rating slot goes.
- `target="_blank" rel="noopener noreferrer"` on all external links per D-02
- `h4` for title to sit below h2 (category) and h3 (tier) in the heading hierarchy
