**Variable replacements (all 13 broken references):**
- `var(--header-title-color)` -> `$global-font-color` (light) / `$global-font-color-dark` (dark) -- used in: `.content h2 color`, `.content h3 color`, `.content > ul > li strong color`, heading selectors
- `var(--global-border-color)` -> `$global-border-color` (light) / `$global-border-color-dark` (dark) -- used in: `.content h2 border-bottom`, `.content > ul > li border`, `.content > hr border-top`, `.content > ul:first-of-type border`
- `var(--global-background-color)` -> `$global-background-color` (light) / `$global-background-color-dark` (dark) -- used in: `.content > ul > li background`, `.content > ul:first-of-type background`
- `var(--single-link-color)` -> `$single-link-color` (light) / `$single-link-color-dark` (dark) -- used in: `.content > ul > li a color`, `.content > ul > li a:hover border-bottom-color`, `.content > ul > li:hover border-color`, `.content > ul:first-of-type li a background`

**File structure (per D-03, D-05, D-06):** Use `.single .content` as base selector (matching theme specificity). Use the flat selector pattern from `_custom.scss` (not nested `[theme=dark] &`). Structure the file in three blocks:

1. **Light mode (default)** -- all styles from the inline block with `var()` replaced by light SCSS variables
2. **Dark mode** -- `[theme=dark] .single .content` selectors for every property that uses a color variable (border-bottom-color, color, background, border-color). Only override color-related properties, not layout/spacing.
3. **Auto theme** -- `@media (prefers-color-scheme: dark) { [theme=auto] .single .content ... }` with identical dark values

**Mobile breakpoint (per LAYOUT-04):** Change `@media (max-width: 768px)` to `@media (max-width: 680px)` to align with LoveIt's mobile header breakpoint from `_core/_media.scss`. Keep the same rules: h2 font-size 1.6rem, list item padding 1.25rem, anchor nav flex-direction column, anchor links display block text-align center.

**Specificity note:** The `.content > ul:first-of-type li a` selector sets `background: $single-link-color` and `color: white`. The dark mode override must set `background: $single-link-color-dark` (the anchor pill background changes in dark mode). The `color: white` stays the same in both modes (white text on colored pill).

**box-shadow rgba values:** The `box-shadow` values (e.g., `rgba(0, 0, 0, 0.05)`) do not reference any broken variables -- keep them as-is in both light and dark mode.

Then add `@import "reading-list";` as the last line of `assets/css/_custom.scss` (per D-03). Hugo's SCSS importer resolves `"reading-list"` to `_reading-list.scss` automatically.

Per D-02: Do NOT define any new CSS custom properties. All values must be SCSS variables resolved at build time.
