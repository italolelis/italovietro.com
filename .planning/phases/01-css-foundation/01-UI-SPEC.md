## SCSS File Contract

Phase 1 produces exactly one new file and one modification:

### New file: `assets/css/_reading-list.scss`

Must contain all styles currently in the `<style>` blocks of both markdown files, rewritten to use SCSS variables instead of broken `var()` references.

**Required SCSS structure:**

```scss
// Reading List styles
// Extracted from content/my-reading-list/index.{en,pt-br}.md
// All var(--*) references replaced with LoveIt SCSS variables

.content {
  // ... light mode styles using $global-* and $single-link-* variables

  h2 {
    border-bottom: 3px solid $global-border-color;
    color: $global-font-color;
    // ...
  }

  // Dark mode override block — matches pattern in _custom.scss
  [theme=dark] & {
    h2 {
      border-bottom-color: $global-border-color-dark;
      color: $global-font-color-dark;
    }
    // ... all dark overrides
  }

  // Auto theme dark mode — matches pattern in _custom.scss
  @media (prefers-color-scheme: dark) {
    [theme=auto] & {
      // ... same dark overrides
    }
  }
}
```

### Modification: `assets/css/_custom.scss`

Add at the bottom:

```scss
@import "reading-list";
```

## Dark Mode Pattern

The exact dark mode pattern from `assets/css/_custom.scss` (lines 28-40) must be replicated for all reading list selectors:

```scss
[theme=dark] .selector { /* dark values */ }

@media (prefers-color-scheme: dark) {
  [theme=auto] .selector { /* same dark values */ }
}
```

This is the LoveIt-standard approach. Do not use CSS custom properties (`var(--*)`). Do not define new CSS custom properties.
