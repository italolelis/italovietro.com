  <action>
Create `layouts/shortcodes/talk.html` following the exact `book.html` pattern (per D-01, D-02).

Parameters to extract with `.Get`:
- `title` (required) -- talk/episode title
- `event` (required) -- conference or podcast name
- `date` (required) -- date string (e.g., "November 2024")
- `type` (required, default "talk") -- one of: talk, podcast, host
- `video_url` (optional, default "") -- link to recording
- `slides_url` (optional, default "") -- link to slides

Body content: `trim .Inner "\n" | .Page.RenderString` for the description/personal note (per D-03 / TALK-03).

HTML structure using BEM naming (per D-02):
```html
<div class="talk-entry talk-entry--{{ $type }}">
  <div class="talk-entry__header">
    <h4 class="talk-entry__title">{{ $title }}</h4>
    <div class="talk-entry__meta">
      <span class="talk-entry__event">{{ $event }}</span>
      <span class="talk-entry__date">{{ $date }}</span>
      {{- if or $videoUrl $slidesUrl }}
      <span class="talk-entry__links">
        {{- if $videoUrl }}<a href="{{ $videoUrl }}" target="_blank" rel="noopener noreferrer">Watch</a>{{- end }}
        {{- if and $videoUrl $slidesUrl }} · {{- end }}
        {{- if $slidesUrl }}<a href="{{ $slidesUrl }}" target="_blank" rel="noopener noreferrer">Slides</a>{{- end }}
      </span>
      {{- end }}
    </div>
  </div>
  {{- if $inner }}
  <div class="talk-entry__description">{{ $inner }}</div>
  {{- end }}
</div>
```

Per D-03: Video and slides links render as small inline links (text "Watch" / "Slides") after the event/date line, separated by a dot. Not buttons or icons.

Per D-04 (Claude's discretion for type-based visual treatment): Use a subtle Font Awesome icon before the title to differentiate types. Use `fa-microphone` for talk, `fa-podcast` for podcast, `fa-headphones` for host. This is lightweight and fits the typography-driven design. The icon goes inside `talk-entry__title` before the text, wrapped in a `<span class="talk-entry__type-icon">`.
