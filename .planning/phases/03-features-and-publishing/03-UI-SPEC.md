## Star Rating Contract

### Shortcode Extension

Add a `rating` parameter to `layouts/shortcodes/book.html`. The rendered HTML inserts a `.book-entry__rating` span **inline after the `<h4>` title element**, within `.book-entry__header`.

**Rendered HTML structure (updated from Phase 2):**

```html
<div class="book-entry book-entry--{type}">
  <div class="book-entry__header">
    <h4 class="book-entry__title">
      <a href="{link}" target="_blank" rel="noopener noreferrer">{title}</a>
      <span class="book-entry__rating" aria-label="{N} out of 5 stars">
        <i class="fas fa-star"></i>
        <!-- repeated N times for rating value -->
      </span>
    </h4>
    <span class="book-entry__author">{author}</span>
  </div>
  <div class="book-entry__description">
    {inner content}
  </div>
</div>
```

**If `rating` param is absent or empty:** Do not render `.book-entry__rating` at all. No empty wrapper.

### Shortcode Hugo Template Logic

```html
{{- $rating := .Get "rating" | default "" -}}
{{- if $rating -}}
<span class="book-entry__rating" aria-label="{{ $rating }} out of 5 stars">
  {{- range (seq (int $rating)) -}}<i class="fas fa-star"></i>{{- end -}}
</span>
{{- end -}}
```

### Star Visual Spec

| Property | Value | Source |
|----------|-------|--------|
| Display | Inline — same line as title, immediately after link text | D-03 |
| Icon | `fas fa-star` (Font Awesome 5 solid star) | D-01 |
| Style | Filled stars only — no empty outlines | D-02 |
| Count | Exactly N filled stars where N = `rating` value (1–5) | D-01 |
| Font-size | 0.75rem (12px) — smaller than title (18px) to not dominate | D-03 |
| Color (light) | `$single-link-color` (`#2d96bd`) | D-04 |
| Color (dark) | `$single-link-color-dark` (`#55bde2`) | D-04 |
| Margin-left | 8px (sm token — between end of title link and first star) | discretion |
| Vertical align | `vertical-align: middle` | discretion |

**Note on star icon spacing:** `letter-spacing` between individual star icons is set to `2px` in the SCSS implementation. This is a typography/rendering property for icon sequences, not a spacing token — it is not part of the 8-point spacing scale.

**Accent reserved for (updated):** book entry title link text color AND star rating icons. Stars use the same accent variable as title links — they are a unified visual signal.

### Rating-to-Tier Mapping

Used when populating `rating="N"` values in content files. Source: D-05.

| Tier | `rating` value |
|------|---------------|
| Must Read | 5 |
| Recommended | 4 |
| Worth Your Time | 3 |
| Newsletters / Podcasts | 4 or 5 (based on description tone — Italo's call per entry) |

**Note:** Tier labels and ratings are independent — tiers remain as manual markdown h3 headings, ratings are per-entry shortcode params. No auto-derivation. Source: D-06.

## Anchor Navigation Contract

### Structure

Anchor nav is a static block at the top of the page content, below the intro paragraph and above the first section. Source: D-08.

```html
<!-- Rendered from markdown ul — existing pattern in content files -->
<nav class="reading-list-nav" aria-label="Jump to section">
  <a href="#currently-reading">Currently Reading</a>
  <a href="#engineering-books">Engineering Books</a>
  <a href="#management-books">Management Books</a>
  <a href="#newsletters">Newsletters</a>
  <a href="#podcasts">Podcasts</a>
</nav>
```

The above is the **semantic intent** — the actual HTML is generated by Hugo from the existing markdown bullet list. Hugo auto-generates `id` attributes on headings that match the link targets. The SCSS targets `ul:first-of-type` (the current Phase 1/2 selector) which styles the anchor nav list.

**"Currently Reading" is the first nav item.** Source: D-07.

### Anchor Nav Visual Style

Style: **text links with separator dots** — clean, typographic, no pill backgrounds. This replaces the Phase 1/2 pill-button style (accent background, border-radius, padding) with a lighter editorial treatment that matches the typography-driven design established in Phase 2.

| Property | Value |
|----------|-------|
| Display | Inline-flex, flex-wrap: wrap, gap: 0 |
| Link style | Plain text links — no background, no border, no border-radius |
| Link color | `$single-link-color` / `$single-link-color-dark` |
| Link font-size | 0.875rem (14px) |
| Link font-weight | 400 |
| Link padding | 0 (no padding on anchor element — separator spacing handled by `::before` pseudo-element) |
| Separator | `::before` pseudo-element: `·` (middle dot, U+00B7), color `$global-font-secondary-color`, padding `0 4px`, on all items except first |
| Hover | `color: $single-link-hover-color` with `transition: color 0.2s ease` |
| Container margin | `margin: 0 0 2rem 0` (32px below, matching xl token) |
| Container padding | 0 (no card background — matches Phase 2 removal of card borders) |
| Container background | none |
| Container border | none |
| Container border-radius | none |

**Phase change from Phase 1/2:** Remove the `.single .content > ul:first-of-type` card styles (background, border, border-radius, padding). Replace with the plain text link separator style above. The pill/card look does not fit the clean editorial design achieved in Phase 2.

**Styles to remove from `_reading-list.scss`:**

```scss
// REMOVE these Phase 1 selectors — replaced by plain anchor nav in Phase 3
.single .content > ul:first-of-type { /* background, border, border-radius, padding: 1.5rem */ }
.single .content > ul:first-of-type li { /* margin, padding */ }
.single .content > ul:first-of-type li a { /* background, color, border-radius, padding, font-weight */ }
.single .content > ul:first-of-type li a:hover { /* transform, box-shadow, opacity */ }
// Plus all dark/auto equivalents
```

### Mobile Anchor Nav

At `max-width: 680px`: separator dots hidden, links stack vertically with `display: block`, `padding: 4px 0`, `text-align: left`.

## Spacing Scale

Inherited from Phase 1 and Phase 2. No new tokens introduced. Star ratings and anchor nav use existing values.

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding |
| sm | 8px | Compact spacing — star `margin-left` (between title text and first star) |
| md | 16px | Default element spacing |
| lg | 24px | Section padding, book entry margin-bottom |
| xl | 32px | Anchor nav margin-bottom, tier h3 margin-top |
| 2xl | 48px | Major section breaks |
| 3xl | 64px | Page-level spacing |

**Phase 3 specific usage notes:**
- Star `margin-left`: 8px (sm token)
- Anchor nav separator padding: `0 4px` (xs token) — applied to `::before` pseudo-element
- Star icon `letter-spacing: 2px` is a typography/rendering property for icon sequences, not a spacing token, and is not part of the scale above

## Typography

All values inherited from Phase 2. Phase 3 adds only the star rating icon size.

**Declared font size scale (4 sizes, unchanged):**

| Role | Size | Weight | Line Height | Element |
|------|------|--------|-------------|---------|
| Tier label / author / anchor nav | 13px (0.8125rem) | 600 / 400 / 400 | 1.2 | Tier h3, `.book-entry__author`, nav links at 0.875rem |
| Body | 16px (`$global-font-size`) | 400 | 1.5 | Page prose, entry descriptions |
| Book entry title | 18px (1.125rem) | 600 | 1.3 | `.book-entry__title a` |
| Section heading (h2) | 32px (2rem) — theme default | 700 | 1.2 | Category headings |

**Star rating icons:** 0.75rem (12px). Intentionally smaller than surrounding text. Not a declared typography size — icon sizing only.

**Anchor nav links:** 0.875rem (14px) — between tier label (13px) and body (16px). Secondary navigation, should read quieter than body prose.

**Note:** The 16px body and 18px book-entry-title are only 2px apart. This is a Phase 2 locked decision inherited here — not changed in Phase 3.
