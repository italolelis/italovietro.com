## Directory Purposes

**`.github/workflows/`:**
- Purpose: GitHub Actions automation
- Contains: `pages.yml` (deploy to GitHub Pages), `pr-checks.yml` (PR validation)
- Key files:
  - `pages.yml`: Deployment workflow triggered on master push
  - `pr-checks.yml`: Build verification for pull requests

**`.devcontainer/`:**
- Purpose: Remote development environment configuration
- Contains: Container definition and VS Code extensions
- Key files:
  - `devcontainer.json`: Container setup (Hugo v1.20, Node.js, extensions)
  - `Dockerfile`: Base container image with Go, Hugo, Node

**`.planning/codebase/`:**
- Purpose: Architecture documentation (this directory)
- Contains: ARCHITECTURE.md, STRUCTURE.md, CONVENTIONS.md, TESTING.md, CONCERNS.md
- Generated by: GSD map-codebase command

**`archetypes/`:**
- Purpose: Content scaffolding templates
- Contains: Default markdown template with basic frontmatter
- Key files: `default.md` - template applied by `hugo new` command

**`assets/`:**
- Purpose: Pre-processed assets
- Contains: SCSS stylesheets and images for pipeline
- Key files:
  - `css/_custom.scss`: Custom styling overrides
  - `css/_override.scss`: Additional overrides
  - `images/`: Images for CSS/template inclusion
  - `music/`: Audio assets

**`content/`:**
- Purpose: All publishable content
- Contains: Markdown files with TOML/YAML frontmatter
- Key files:
  - `content/posts/`: Blog articles (each post in own directory with `index.en.md`, `index.pt-br.md`)
  - `content/talks/`: Speaking engagements page
  - `content/my-reading-list/`: Books and resources
  - `content/_index.en.md`, `content/_index.pt-br.md`: Homepage
  - `content/tags/`, `content/categories/`: Taxonomy overrides (optional)

**`layouts/`:**
- Purpose: Custom Hugo template overrides
- Contains: HTML templates using Go template syntax
- Key files:
  - `layouts/_default/`: Default page layouts (inherits from theme)
  - `layouts/_default/_markup/`: Custom markdown rendering (code blocks, mermaid diagrams)
  - `layouts/talks/single.html`: Custom talk page layout with featured image
  - `layouts/taxonomy/term.html`: Tag/category listing page
  - `layouts/partials/header.html`: Navigation and search header (desktop + mobile)
  - `layouts/partials/plugin/`: Plugin components (analytics, image handling)

**`public/`:**
- Purpose: Generated static site
- Contains: Complete HTML, CSS, JS output from build
- Generated: By `hugo` CLI
- Committed: No (ignored in .gitignore)

**`resources/`:**
- Purpose: Hugo resource cache
- Contains: Processed CSS, images, and other compiled assets
- Generated: By Hugo during build
- Committed: No (ignored in .gitignore)
- Caching: GitHub Actions caches between builds for faster builds

**`static/`:**
- Purpose: Static files served as-is
- Contains: Favicons, web manifest, redirects, logo, images
- Key files:
  - `images/logo.svg`: Site logo
  - `images/speaking.webp`: Speaking/about page image
  - `favicon.ico`, `favicon-16x16.png`, etc.: Favicon variants
  - `site.webmanifest`: PWA web app manifest
  - `_redirects`: Netlify/edge redirect rules (if hosted there)

**`themes/LoveIt/`:**
- Purpose: Theme submodule with base layouts and styles
- Contains: Default Hugo theme implementation
- Key subdirectories:
  - `themes/LoveIt/layouts/`: Default layouts inherited by custom `layouts/`
  - `themes/LoveIt/static/`: Theme assets
  - `themes/LoveIt/assets/`: Theme style and script inputs
- Committed: Yes (as git submodule)
- Customization: Extended via `layouts/` (same directory structure overrides theme)

## Naming Conventions

**Files:**
- `index.[lang].md`: Content files (e.g., `index.en.md`, `index.pt-br.md`)
- `single.html`: Template for individual page display
- `term.html`: Template for taxonomy list pages
- `_*.scss`: SCSS partial files (convention: underscore prefix)
- `*.yaml`, `*.yml`: YAML configuration files
- `.md`: All documentation files

**Directories:**
- Lowercase with hyphens: `my-reading-list`, `building-my-homelab` (content slug directories)
- Lowercase with hyphens: `cto-reading-list-1`, `do-job-titles-matter` (post directories)
- Lowercase: `archetypes`, `layouts`, `content`, `assets`, `static`, `themes`
- Dot-prefix for hidden/tooling dirs: `.github`, `.devcontainer`, `.vscode`, `.planning`

**Classes/IDs in templates:**
- kebab-case: `header-desktop`, `menu-item`, `single-title`, `archive-item-list`
- Prefix by concern: `search-*`, `menu-*`, `theme-*`, `language-*`

## Where to Add New Code

**New Blog Post:**
- Create directory: `content/posts/[slug]/`
- Create files: `index.en.md`, `index.pt-br.md` (for bilingual posts)
- Required frontmatter: title, date, draft, author, description, tags, categories
- Optional: featuredimage, resources
- Template applied: Theme default post layout (inherited from `themes/LoveIt/layouts/_default/single.html`)

**New Page Type:**
- Custom layout: `layouts/[type]/single.html` (replaces theme layout for content with `type: [type]` in frontmatter)
- Use data from: `.Title`, `.Content`, `.Params`, `.Site.Params`
- Include partials: `{{ partial "header.html" . }}`, `{{ partial "footer.html" . }}`
- Example: `layouts/talks/single.html` shows featured image custom layout

**Custom CSS:**
- Add to: `assets/css/_custom.scss`
- Pattern: SCSS syntax, scoped selectors, extends theme variables
- Processing: Automatically compiled by Hugo asset pipeline + PostCSS
- Variables: Theme provides color, typography, spacing variables via `themes/LoveIt/assets/scss/`

**Partials/Components:**
- Create: `layouts/partials/[component-name].html`
- Call syntax: `{{ partial "component-name.html" . }}`
- Pass context: `.` provides full page context to partial
- Organization: Group related partials in subdirectories (e.g., `layouts/partials/plugin/`)

**Static Assets:**
- Non-processed: `static/` (images, manifest, favicon)
- Processed by asset pipeline: `assets/` (CSS, images referenced in styles)
- Distinction: Use `assets/` if processing needed (SCSS compilation, optimization); `static/` for as-is files

**Markdown Rendering:**
- Code block theme: `layouts/_default/_markup/render-codeblock.html`
- Diagram support: `layouts/_default/_markup/render-codeblock-mermaid.html`
- Link rendering: `layouts/_default/_markup/render-link.html` (if needed)

## Special Directories

**`public/`:**
- Purpose: Build output
- Generated: Yes (by `hugo` command)
- Committed: No (.gitignore entry)
- Contents: Complete static HTML/CSS/JS site

**`resources/`:**
- Purpose: Hugo processing cache
- Generated: Yes (by Hugo during build)
- Committed: No (.gitignore entry)
- Contents: Processed assets, cached resources

**`themes/LoveIt/`:**
- Purpose: Base theme submodule
- Generated: No (git submodule)
- Committed: Yes (as submodule reference in `.gitmodules`)
- Modification: Do NOT edit directly; create overrides in `layouts/` instead

**`.claude/`:**
- Purpose: Claude Code workspace data
- Generated: Yes (by Claude)
- Committed: No (.gitignore entry)

**`.git/` and `.github/`:**
- Purpose: Git version control and GitHub-specific configuration
- Generated/Committed: Yes
- `.github/workflows/`: Actions automation (see CI/CD section)

## Build Outputs

**Development Server:**
```bash
hugo server -D
# Output: Live server at http://localhost:1313 with draft posts
```

**Production Build:**
```bash
hugo --gc --minify --baseURL "https://italovietro.com/"
# Output: Minified static files in public/
# Flags: --gc (garbage collect), --minify (compress output)
```

**PR Check Build:**
```bash
hugo --gc --minify
# Output: Same as production but without baseURL override
```
